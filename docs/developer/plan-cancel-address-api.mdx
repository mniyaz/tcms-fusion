# Cancel Address API

Remove a pickup or delivery address from a booking. The primary pickup (`Pickup 1`) and primary delivery (`Delivery 1`) cannot be canceled.

## Endpoint {#endpoint}

```
POST {BASE_URL}/rest/planner/cancelAddress/{userId}/{authKey}
```

| Item | Value |
|------|--------|
| **Method** | POST |
| **Path** | `/rest/planner/cancelAddress/{userId}/{authKey}` |
| **Content-Type** | application/json |

## Path parameters {#path-parameters}

| Parameter | Type | Description |
|-----------|------|-------------|
| `userId` | string | User identifier (e.g. `user123`). |
| `authKey` | string | Authentication key for the user. |

## Request body {#request-body}

JSON object with the address to cancel:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | `"PickupAddress"` or `"DeliveryAddress"`. |
| `id` | number | Yes | Address ID (e.g. `456`). |
| `title` | string | Yes | Address title (e.g. `"Delivery 2"`, `"Pickup 2"`). |
| `goodsDescription` | array | Yes | Array of goods description objects for this address. Send existing data if available; otherwise send `[]`. This key is mandatory. |

`goodsDescription` array items (GoodsDescription object):

| Field | Type | Description |
|-------|------|-------------|
| `itemCode` | string | Item code. |
| `dimensionLength` | number | Length dimension. |
| `dimensionWidth` | number | Width dimension. |
| `dimensionHeight` | number | Height dimension. |
| `dangerous` | boolean | Whether the goods are dangerous. |
| `delivered` | boolean | Whether goods were delivered. |
| `pickupId` | string | Pickup reference (e.g. `"O1"`). |
| `type` | string | `"GoodsDescription"`. |
| `packageType` | string | Package type (e.g. `"Pallet"`). |
| `deliveryRefId` | number | Delivery reference ID. |
| `pieces` | number | Number of pieces. |
| `deliveryId` | string | Delivery reference (e.g. `"D1"`). |
| `codReceived` | boolean | COD received flag. |
| `picked` | boolean | Whether goods were picked. |
| `cube` | number | Cube/volume. |
| `goodsDescription` | string | Description text (e.g. `"GENERAL GOODS"`). |
| `shipRefNo` | string | Shipment reference number. |
| `stackable` | boolean | Whether goods are stackable. |
| `gdId` | number | Goods description ID. |
| `weight` | number | Weight value. |
| `customerName` | string | Customer name. |
| `disposable` | boolean | Disposable flag. |
| `bookingId` | string | Booking ID. |
| `cubeType` | string | Cube unit (e.g. `"M3"`). |
| `coNumber` | string | CO number. |
| `weightType` | string | Weight unit (e.g. `"KG"`). |
| `createdDate` | string | Creation date. |
| `cod` | string | COD amount or value. |
| `pickupRefId` | number | Pickup reference ID. |

### Example (cancel delivery address) {#example-cancel-delivery-address}

```json
{
  "type": "DeliveryAddress",
  "id": 456,
  "title": "Delivery 2",
  "goodsDescription": []
}
```

```json

  "type": "PickupAddress",
  "id": 789,
  "title": "Pickup 2",
  "goodsDescription": []
}
```

## cURL example {#curl-example}

```bash
curl -X POST "{BASE_URL}/rest/planner/cancelAddress/user123/auth_key_xyz" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "DeliveryAddress",
    "id": 456,
    "title": "Delivery 2",
    "goodsDescription": []
  }'
```

Replace `{BASE_URL}`, `user123`, and `auth_key_xyz` with your base URL, user ID, and auth key.

## Responses {#responses}

### 200 OK {#200-ok}

Address was canceled successfully.

**Body:**

```json
{"success": true}
```

### 400 Bad Request {#400-bad-request}

Invalid or disallowed request.

**Body (examples):**

- `{"error": "Invalid Address Data!"}` – Request body could not be parsed or converted to a valid address, or required field `goodsDescription` is missing.
- `{"error": "Cannot Cancel Primary Pickup Address!"}` – The address title is `"Pickup 1"` (primary pickup cannot be removed).
- `{"error": "Cannot Cancel Primary Delivery Address!"}` – The address title is `"Delivery 1"` (primary delivery cannot be removed).
- `{"error": "Unsupported Address Type!"}` – `type` is not a supported address type.

### 401 Unauthorized {#401-unauthorized}

Invalid or missing `userId` / `authKey` for the tenant.

**Body:** Empty.

### 500 Internal Server Error {#500-internal-server-error}

Server error while processing the request.

**Body:**

```json
{"error": "<error message>"}
```

## Rules {#rules}

- **`goodsDescription`** is mandatory. For pickup and delivery addresses, send the existing array of goods descriptions if available; otherwise send `[]`.
- **Primary pickup** (`title` = `"Pickup 1"`) cannot be canceled; the API returns 400.
- **Primary delivery** (`title` = `"Delivery 1"`) cannot be canceled; the API returns 400.
- Only non-primary pickup and delivery addresses can be removed.
- Authentication is validated per tenant; invalid `userId`/`authKey` returns 401.
